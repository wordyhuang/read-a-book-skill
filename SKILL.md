# 书籍拆解技能 — Read a Book Skill

基于赵周《这样读书就够了》RIA 拆书法，按「选书评估 → RIA 拆解 → 拆为己用」三阶段，对非虚构书籍进行以应用为导向的结构化拆解。

---

本技能设计为**与具体 Agent 平台解耦**，任何具备以下基础能力的 AI Agent 均可直接使用：

## 核心理论

### RIA 拆书法（赵周）

赵周《这样读书就够了》提出的核心方法论，强调**阅读本身不能创造价值，改变行为才有**。拆书包括三个核心环节：

```
R — Reading（阅读拆页）
    └─ 从书中提取核心片段、知识点或方法论
    │
I — Interpretation（重述引导）
    └─ 用自己的语言精准复述原内容，加深理解
    │
A — Appropriation（拆为己用）
    ├─ A1：关联经验——回想自己或他人相关的经历、案例
    └─ A2：规划运用——制定具体的行动方案，把知识用起来
```

### 成人学习五大公理

RIA 拆书法的底层理论依据：

1. **自我导向** — 学习者主动选择学习内容，结合自身问题与目标
2. **关联经验** — 学习必须以学习者已有的经验为起点和参照
3. **强调实践** — 学习的最终目的是解决实际问题、改变行为
4. **聚焦具体问题** — 以问题为中心组织学习，而非以知识为中心
5. **内在驱动** — 内生动力（解决问题/提升能力/自我实现）比外部压力更持久

### 核心理念

- **拆书不为记，拆书为用** — 知识只有转化为行动才有价值
- **拆为己用** — 不是拆给作者看，而是拆给自己的实际生活和工作
- **先拆后整** — 不追求逐章通读，从最触动你的片段开始拆解
- **输出倒逼拆解** — 只有能教给别人、能写下来的，才是真正拆透了的

## 1. 核心工作流 — RIA 拆书流程

RIA 拆书法不按书籍分类走分支，而是统一走「选书评估 → R 阅读拆页 → I 重述引导 → A1 关联经验 → A2 规划运用 → 总结输出」六阶段流程。本书类型只影响拆页的关注侧重点，不影响流程结构。

## 2. Phase 0：选书评估

通过 Agent 可用的**网络搜索 + 网页内容获取**工具，获取以下信息，判断书籍的拆解价值：

| 来源 | 用途 | 优先级 |
|------|------|--------|
| 豆瓣读书页面 | 书籍元数据、评分、目录、短评 | 第一 |
| 豆瓣读书笔记/书评 | 原文摘录、章节概括、读者反馈 | 第二 |
| 得到/微信读书/简书笔记 | 详细章节拆解笔记 | 第三 |
| 百度百科/维基百科 | 作者背景、书籍概况 | 第四 |

**必须采集的字段**：
- ✅ 书名（完整中英文）
- ✅ 作者（全名、背景简介）
- ✅ 出版社、出版时间、ISBN
- ✅ 扉页/封面语
- ✅ 全书目录结构
- ✅ 序言/前言内容（作者写作初衷、目标读者、核心问题）
- ✅ 适用人群判断（什么人最需要这本书）
- ✅ 前置知识要求（零门槛/基础入门/专业背景/学术研究）

### 拆解价值评估

| 评估维度 | 高价值信号 | 低价值信号 |
|:--------:|:---------|:---------|
| **问题匹配** | 解决具体问题、行动导向 | 纯理论、重叙事轻方法 |
| **可拆密度** | 每章有独立概念/方法/框架 | 连续叙事、缺少可截取的知识点 |
| **适用广度** | 经验可迁移到多个场景 | 高度专业、场景单一 |
| **内容可获得性** | 公开资料丰富（书评/笔记/目录齐全） | 冷门书、公开信息极少 |

默认可拆解，但当以上四个维度全部指向低价值时，建议用户考虑其他书籍。

## 3. Phase 1-4：RIA 拆解规格

RIA 拆书法统一处理所有非虚构书籍，不按类型分分支。以下为各阶段的详细规格。

---

### Phase 1：R — 阅读拆页

对书中的每个核心知识点/片段提取以下内容：

| 要素 | 说明 | 示例（《精进》第一章） |
|:----:|:----:|:---------------------:|
| **核心片段** | 该片段要解决什么问题、提出什么主张 | 采铜法则：收益值×半衰期 |
| **原文精华** | 2-3 段代表性原话，严格标注来源 | "尽量少做短半衰期的事情" |
| **关键词** | 该片段的核心术语 | "郑重""半衰期""远期未来" |
| **归因判断** | 这个概念属于：方法/概念/模型/案例/观点 | 方法+模型 |

#### 不同书籍类型在 R 阶段的关注侧重

| 书籍类型 | 拆页关注重点 | 示例 |
|:--------:|:-----------|:----|
| 实用/工具类 | 方法步骤、行动框架、清单 | 采铜法则的适用条件 |
| 认知/科普类 | 概念定义、实验证据、结论 | 系统1和系统2的核心区别 |
| 商业/管理类 | 商业模型、数据、案例成败 | 垄断 vs 竞争的判断标准 |
| 哲学/思想类 | 核心命题、论证结构 | "学而时习之"的完整论证 |

---

### Phase 2：I — 重述引导

对 R 阶段提取的每个核心片段，按以下结构重述：

| 要素 | 说明 | 标准 |
|:----:|:----|:----|
| **用自己的话重述** | 不说原话，用你自己的语言把作者的意思说清楚 | 让没读过书的人也能听懂 |
| **补充说明** | 对比/举例/打比方，帮助理解 | 用日常生活中的类比 |
| **你与它的关系** | 这个点为什么值得你关注 | 解决什么问题、填补什么盲区 |

**I 阶段质量检查**：
如果重述后别人能准确复述核心意思，则 I 过关。如果别人听完还是含糊的，重述不达标。

**示例（《精进》采铜法则 — I 重述）**：

> 采铜说，判断一件事值不值得做，可以从两个维度来打分：一个是做这件事当下能带来多大满足感，一个是这个满足感能持续多久。他把前者叫收益值，后者叫半衰期。真正值得多做的，不是收益值高的，而是半衰期长的——也就是做完之后效果还在持续的事。

---

### Phase 3：A1 — 关联经验

对每个拆页，关联自己或他人的真实经验：

| 要素 | 说明 | 示例（《精进》采铜法则） |
|:----:|:----|:------------------------|
| **相关经历** | 你或身边的人有没有类似的经验？ | "我每天刷短视频两小时，快乐到关屏幕就消失" |
| **验证对比** | 你经历的事跟作者说的一致还是矛盾？ | "我以前觉得刷短视频是在放松，但按采铜的标准这属于短半衰期" |
| **原因反思** | 从作者的角度回头看，你当时做对了什么/做错了什么？ | "原来我一直在做短半衰期的事却不自知" |
| **规律提炼** | 从具体案例中能抽象出什么通用经验？ | "凡是用'打发时间'作为理由做的事，大概率都是短半衰期" |

**经验来源参考**（按优先级）：
1. 个人亲身经历
2. 身边人的真实案例
3. 公开知名案例（企业/名人/历史事件）
4. 虚拟但合理的场景构建

---

### Phase 4：A2 — 规划运用

对每个拆页，制定具体行动方案：

| 要素 | 说明 | 示例（《精进》采铜法则） |
|:----:|:----|:------------------------|
| **行动目标** | 具体要做什么？用 SMART 描述 | "本周每天睡前用采铜法则评估当天做的三件事" |
| **启动时间** | 什么时候开始第一步？ | "今晚就开始" |
| **验证标准** | 做到什么程度算用上了？ | "连续7天，每天记录半衰期评分" |
| **障碍预判** | 什么可能会阻止你做这件事？ | "一累就会忘" |
| **对策预案** | 遇到障碍怎么办？ | "把评估贴纸粘在枕头上" |

A2 质量检查：
- 如果听完行动方案后什么都不清楚→太模糊，需要细化
- 如果听完马上知道自己今晚可以做什么→过关
- 如果方案涉及其他人（需要配合/花钱/等条件）→太依赖外部，建议回到自己能独立启动的最小动作

## 6. 关键约束

1. **原文引用必须忠实于原书**：所有引用的原话必须标注来源（章节/页数或来源平台），不得编造原文
2. **RIA 三环节缺一不可**：不能只有 R 没有 AI，也不能只有 I 没有 A。拆书的最终目的是用
3. **A2 必须可执行**：行动方案要具体到今晚/明早能做什么，不能停留在"我要努力"级别的空话
4. **I 重述必须用自己的话**：不能是原文的缩写或改写，必须是你理解后的自然表达
5. **A1 关联要有真实感**：优先用亲身经历，其次是身边案例。编造的经历比不写更糟糕
6. **多拆页处理原则**：如果一本书有多个值得拆的点，每个点独立走 R→I→A1→A2，不要混在一起
7. **冷门书籍说明**：当公开资料不足以完成完整拆解时，应在报告中说明局限，并建议用户自行补充阅读
8. **信息标注**：在报告末尾注明信息来源（如"本文档基于豆瓣读书标注、公开书评摘录及原书公开信息整理"）
9. **去AI味处理**：除原书原文引用外，所有生成的内容（重述、关联、规划等）必须按 [remove-ai-flavor-writing-skill](https://github.com/B1lli/remove-ai-flavor-writing-skill) 的规则去除AI写作痕迹，参考下方「去AI味速查表」

### 去AI味速查表

输出前逐项检查，确认不存在以下模式：

| 类别 | 高危险型 | 改写方向 |
|:----:|:---------|:---------|
| **二元对比壳** | `不是...而是` `并非...而是` `不在于...而在于` `不只是...更是` `与其说...不如说` | 如果 A 是铺垫，直接陈述 B；如果 A 和 B 都需要，写成具体关系而非说教式纠正 |
| **分步推进壳** | `先...再...` `第一步...第二步...` `从...到...` | 顺序改变结果才保留，否则直接写动作本身 |
| **本质断言** | `真正重要的是` `本质上` `核心在于` `底层逻辑` `关键就在于` | 直接说出主题本身，用事实/因果替代抽象强调 |
| **助手指路牌** | `下面我们来` `接下来` `我们可以看到` `总的来说` `值得注意的是` `说白了` `划重点` `拆一拆` | 直接进入内容，让内容自己推进 |
| **简化断言** | `很简单：` `答案很简单：` | 替换为具体条件、证据或决策过程 |
| **模版标点** | `概念：解释` `问题：答案` 等冒号模板句 | 只在自然需要的地方用冒号 |
| **过度平行** | 三段以上结构、长度、情绪一致的分句 | 砍掉一段，一段展开成例子，一段改为平实陈述 |
| **空结尾反问** | `你觉得呢？` `是不是很有启发？` `你有没有类似经历？` | 用事实、场景、后果收尾，或去掉 |

## 7. 输出格式 B — 口播稿

### 7.1 设计意图

在标准结构化报告（Section 5）之外，提供一种**可直接朗读的连续稿件格式**。适用于播客、音频节目、视频口播、读书直播等以听为主的场景。

### 7.2 转换规则

当用户要求"播报稿""朗读版""音频稿""连续输出"或明确指定使用本格式时，将原拆解内容按以下规则转换。**注意：转换完成后，必须执行去AI味检查（见 6. 去AI味速查表），确保口播稿中不存在AI痕迹。**

| 原结构化元素 | 转换方式 |
|:-----------:|---------|
| `#` / `##` / `###` 标题 | 转换为自然过渡句，用事实推进而非助手指路牌（不用"接下来我们来看看"这类） |
| `-` 列表 / 表格 | 转换为连续叙述，用"其中""包括""比如"引导 |
| `**加粗**` 强调 | 保留加粗，作为口语重音指示 |
| `>` 引用块 | 用"书里说""作者原话是"等自然引导 |
| 编号/序数词 | 用自然顺序表达，避免"第一、第二"的机械列举 |
| `{代码框}` 公式 | 用平实语言转述 |

### 7.3 核心写作原则

1. **无分段结构** — 不使用标题、列表、表格等结构化标记
2. **无旁白说明** — 不使用"（笑）""（停顿）""（主持人）"等舞台指示
3. **无显性编号** — 不使用"第一""第二""首先""其次""1."等机械分点
4. **自然过渡** — 段落之间用承上启下的语句连接，而非标题切换
5. **口语化** — 短句为主，使用"你"而非"读者"，避免长定语嵌套
6. **情绪节奏** — 开头有吸引力（设问/故事/反常识断言），中间有起伏，结尾有余韵
7. **开场有引、收尾有回** — 稿件最开头必须有问候+引出话题的**开场白**，最末尾必须有结束+回应的**结尾语**，两端形成完整的播报闭环
8. **去AI味** — 口播稿是人讲给人听的，不是AI写给AI读的。写完逐项检查去AI味速查表，清除二元对比壳、本质断言、助手指路牌、空结尾反问等AI常见模式

### 7.4 信息层次融合表

每一份连续内容稿件至少覆盖以下信息层次，但**全部以自然叙述方式呈现**：

| 信息层次 | 融合方式 |
|:--------:|---------|
| 个性开场 + 目标人群引导 | 第一段用"我是XX"建立身份+以"如果你是{目标人群}，{痛点}"的句式锁定听众 |
| 书籍元数据（书名/作者） | 紧接着自然带出 |
| 核心议题/全书主题 | 在前 1/3 处清楚点明 |
| 关键概念 | 在对应位置用通俗方式解释，避免生硬抛术语 |
| 实质案例/原文精华 | 用"书里提到""作者说"等自然引导 |
| 精髓提炼/核心洞见 | 在稿件高潮处着重呈现，适当放慢节奏 |
| 批判性审视 | 收尾时自然融入，不单独成段 |

### 7.5 写作模板

```
━━━ 开场白 ━━━
{个性化问候：如"各位朋友大家好，我是你们的老朋友XX"}
{目标人群 + 痛点铺开：一句话勾勒目标人群，快速带出书中能解决的问题}
{引出书名+作者}
{开篇钩子：一个反常识断言 / 引人深思的问题 / 生动的故事场景}

自然地引出全书核心议题——这本书在探讨什么，为什么值得关注

逐章或按话题群组介绍关键内容：
  - 每个概念用日常语言解释
  - 引用案例或实验时用"书中的XX故事说""作者指出"
  - 关键结论处适当放慢节奏

全书的精髓提炼——最值得记住的观点 / 公式 / 模型

批判性审视——哪些观点有局限、哪些需要结合个人情况判断

自然过渡到结语

━━━ 结尾语 ━━━
{对整期内容的简短回顾}
{开放性余韵 / 引人深思的问题 / 鼓励行动的一句话}
{结束语：如"感谢你的收听，我们下期再见"}

━━━ 去AI味检查 ━━━
▢ 无二元对比壳（不是...而是/并不在于...而在于）
▢ 无本质断言（真正重要的是/本质上/核心在于）
▢ 无分步推进壳（先...再...）
▢ 无助手指路牌（总的来说/值得注意的是/划重点）
▢ 无简化断言（很简单：/答案很简单：）
▢ 无模板标点（概念：解释）
▢ 无空结尾反问
▢ 段落长短不一，节奏有起伏
```

### 7.6 写作示例（基于《精进》第一章，含开场白和结尾语，已去AI味）

各位朋友大家好，我是你们的老朋友XX。如果你是创业者、管理者，或者一个正在快速变化里追赶的人——今天这本书能帮你找到那个卡住你的点。它叫《精进》，作者采铜，浙江大学心理学博士。他在扉页上写了一句很扎心的话——盲目的努力，只是一种缓慢的叠加。采铜用一种很特别的方式开了头，他讲了老家的一个挂钟。每次回家看到那钟，他就觉得时间走得很慢，很放松，稿子没写就没写吧，慢慢来。但用闹钟看时间的时候，就觉得时间走得好急。他把这个叫"母亲的挂钟效应"——你对时间的感觉不是客观的，心理状态和环境会改变它。焦虑的时候时间紧迫，放松的时候时间变慢。这引出了第一章的核心问题：你跟时间的关系，决定了你怎么度过这一生。

采铜提了一个叫"郑重"的概念，说郑重不是板着脸做计划，而是一种对当下事情全身心投入的状态，不敷衍、不迟疑、不摇摆。然后他给了一个判断框架，叫"采铜法则"，看两样：一件事能带来多大收益，收益能持续多久。他建议多做事后效果还能持续很久的事，哪怕单次收益不高。每天背十首宋词，单次收益不大，一年下来底蕴就不一样了。反过来，那些做的时候很快乐、关上屏幕什么都不剩的事——比如刷短视频——能少就少。下次纠结做什么的时候，问问自己：做完之后，效果明天还在吗？以上就是今天关于时间管理的内容，如果对你有用，就从今晚开始做一件长半衰期的小事吧。感谢收听，下期再见。
