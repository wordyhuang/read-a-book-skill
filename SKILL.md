# 书籍拆解技能 — Read a Book Skill

基于 Adler《如何阅读一本书》分析阅读理论，按「扉页保留 → 逐章原文提取 → 精髓提炼」三步骤，对实用/工具、认知/科普、商业/管理、哲学/思想四类非虚构书籍进行结构化拆解。

---

## 0. Agent 适配说明

本技能设计为**与具体 Agent 平台解耦**，任何具备以下基础能力的 AI Agent 均可直接使用：

### 所需能力清单

| 能力 | 用途 | 是否必须 |
|:----:|:----:|:--------:|
| **网络搜索** | 通过书名/作者检索书籍元数据、目录、书评等公开资料 | ✅ 必须 |
| **网页内容获取** | 读取豆瓣、百科、书评网站等页面的详细内容 | ✅ 必须 |
| **文件读取** | 读取本地已存在的拆解报告、案例参考等素材 | ✅ 必须 |
| **文件写入** | 将生成的拆解报告或口播稿保存到本地文件 | ✅ 必须 |

### 平台适配方式

不同 Agent 平台调用本技能时，按以下方式映射能力：

| 本技能中的能力描述 | TRAE/SOLO | Claude | ChatGPT / GPTs | 通用泛化 |
|:-----------------:|:---------:|:-----:|:--------------:|:--------:|
| 网络搜索 | `WebSearch` | `web_search` / `search` | 联网搜索（Bing API） | 任意能返回搜索结果的工具 |
| 网页内容获取 | `WebFetch` | `web_fetch` / `fetch` | 浏览器 / 网页阅读器 | 任意能读取 URL 内容的工具 |

## 核心理论

### 阅读层次
- **检视阅读**：系统化略读（书名→目录→索引→关键章节），15-30 分钟掌握全书框架
- **分析阅读**：15 条规则，彻底吃透一本书
- **主题阅读**：多本书横向比较，构建知识体系

### 主动阅读四问
1. 这本书到底在谈什么？— 主题与框架
2. 作者细说了什么，怎么说的？— 内容与论证
3. 这本书说得有道理吗？— 批判性评价
4. 这本书跟你有什么关系？— 意义与应用

## 2. Phase 1：信息采集（全局统一）

通过 Agent 可用的**网络搜索 + 网页内容获取**工具，获取以下信息来源：

| 来源 | 用途 | 优先级 |
|------|------|--------|
| 豆瓣读书页面 | 书籍元数据、短评、目录信息 | 第一 |
| 豆瓣读书笔记/书评 | 原文摘录、章节概括 | 第二 |
| 得到/微信读书/简书笔记 | 详细章节拆解笔记 | 第三 |
| 百度百科/维基百科 | 作者背景、书籍概况 | 第四 |
| 长篇书评/读书笔记 | 案例细节、论证逻辑 | 补充 |

**必须采集的字段**：
- ✅ 书名（完整中英文）
- ✅ 作者（全名、背景简介）
- ✅ 出版社、出版时间、ISBN
- ✅ 扉页/封面语（书封或扉页上的核心一句话）
- ✅ 序言/前言内容（尤其作者自述写作初衷、目标读者）
- ✅ 完整目录（各章标题 + 各章内子标题）
- ✅ 全书核心论题（序言中明确提出的问题）

## 3. Phase 2：书籍画像（全局统一）

### 维度A：类型判定

根据已采集的信息，按以下优先级判定书籍类型：

| 判断依据 | 判定方法 |
|----------|----------|
| 书名暗示 | 含"方法/技巧/习惯/指南/精进" → 实用类；含"原理/科学/大脑/认知" → 认知类 |
| 目录关键词 | 含"怎么做/步骤/清单/行动" → 实用类；含"实验/数据/研究" → 认知类 |
| 作者背景 | 学术界/科学家 → 认知类；企业家/咨询师 → 商业类；哲学家 → 哲学类 |
| 介绍/序言 | 明确说"教你如何" → 实用类；"探讨什么是什么" → 认知类 |
| 兜底 | 以上模糊时默认为实用类 |

### 维度B：适用人群判定

#### B1 目标读者画像

| 判断线索 | 来源 | 典型表述 |
|----------|------|----------|
| 作者自述目标读者 | 序言/前言 | "写给XX的人""献给XX" |
| 书名暗示 | 书名/副标题 | "年轻人的第一本XX" |
| 话题领域 | 目录+介绍 | 职场/管理→职场人；学习法→学生/职场人 |
| 语言难度 | 样章判断 | 通俗→大众读者；专业术语密集→从业者/研究者 |

#### B2 前置知识要求（4 级刻度）

| 等级 | 描述 | 示例书 |
|:---:|:----:|:------:|
| **零门槛** | 无需任何专业知识，普通大众即可阅读 | 《精进》《习惯的力量》 |
| **基础入门** | 需要对该领域有基本概念即可 | 《思考，快与慢》 |
| **专业背景** | 需要具备该领域的系统性知识储备 | 《存在与时间》 |
| **学术研究** | 需要对该领域的文献和研究方法有深入了解 | 《纯粹理性批判》 |

#### B3 阅读目的匹配

| 阅读目的 | 匹配信号（目录+序言关键词） | 适合的书籍类型 |
|:--------:|:--------------------------:|:--------------:|
| **解决具体问题** | "怎么做""步骤""方法""技巧" | 实用/工具类 |
| **理解世界/人** | "原理""机制""为什么""真相" | 认知/科普类 |
| **提升商业认知** | "商业模式""竞争""增长""战略" | 商业/管理类 |
| **获得思想启迪** | "什么是""本质""意义""追问" | 哲学/思想类 |
| **培养技能** | "练习""训练""刻意""能力" | 实用/技能类 |

#### B4 阅读收益预期

总结读者读完本书后能获得的核心价值（2-4 条）。

#### B5 合并输出

将 B1-B4 合并，放在「一、书籍画像」的"适用人群"子板块中。

## 4. Phase 3-5：各类型分支规格

类型判定后，后续三个阶段根据类型走不同分支。

---

### 分支 A：实用/工具类

**典型书籍**：精进、高效能人士的七个习惯、刻意练习

#### Phase 3-A：逐章拆解

对每一章提取以下内容：

| 要素 | 说明 | 示例（《精进》第一章） |
|:----:|:----:|:---------------------:|
| **方法步骤** | 作者给出的具体操作步骤/流程/框架 | 采铜法则：收益值×半衰期 |
| **案例验证** | 书中用来证明方法有效的案例 | 母亲的挂钟故事 |
| **核心概念** | 定义的术语/模型 | "郑重""远期未来""近期未来" |
| **原文精华** | 2-5 段最具代表性原话，优先选含建议/方法/总结的段落 | "尽量少做短半衰期的事情……" |

#### Phase 4-A：精髓提炼

- **核心洞见**：一句话金句（最精炼的总结）
- **可操公式**：`{方法名} = {步骤1} → {步骤2} → {步骤3}` 或一个可填写的框架表格

**示例（《精进》第一章）**：
- 核心洞见：**郑重地对待时间，把精力投入长半衰期的事情上**
- 可操公式：`价值判断 = 收益值(高/低) × 半衰期(长/短) → 长半衰期的事多做`

#### Phase 5-A：总结输出

1. 全书的行动清单（按优先级排列）
2. 章节间的递进逻辑
3. 适用场景边界（适合什么人、什么阶段）
4. 对比阅读推荐（同类方法书）

---

### 分支 B：认知/科普类

**典型书籍**：思考，快与慢、人类简史、枪炮病菌与钢铁

#### Phase 3-B：逐章拆解

| 要素 | 说明 | 示例（《思考，快与慢》第二章） |
|:----:|:----:|:--------------------------:|
| **核心概念** | 该章提出的核心概念及其定义 | 系统1（快思考）、系统2（慢思考） |
| **实验/证据** | 支持该概念的实验、数据、案例 | 琳达问题、球拍实验 |
| **思维模型** | 作者的论证模型/框架图 | 双系统模型图 |
| **原文精华** | 2-5 段原话，优先选含定义/论证/结论的段落 | "系统1和系统2……" |

#### Phase 4-B：精髓提炼

- **核心洞见**：一句金句（通常是作者的结论性陈述）
- **可操公式**：`{概念} = {前提条件} → {机制} → {结果}` 或一个因果关系图

**示例（《思考，快与慢》第二章）**：
- 核心洞见：**人的大脑有两套决策系统，多数错误来自系统1的直觉捷径**
- 可操公式：`决策质量 = 任务难度 × 系统激活(1或2) — 认知放松度`

#### Phase 5-B：总结输出

1. 全书的概念网络图（概念间的关联）
2. 最关键的 3-5 个核心结论
3. 该理论体系的应用场景
4. 批判性审视：争议/局限/后续研究

---

### 分支 C：商业/管理类

**典型书籍**：从0到1、创新者的窘境、蓝海战略

#### Phase 3-C：逐章拆解

| 要素 | 说明 | 示例（《从0到1》第三章） |
|:----:|:----:|:---------------------:|
| **商业模型** | 该章提出的模型/框架/矩阵 | 垄断 vs 竞争 |
| **数据支撑** | 关键数据、统计、调研结果 | 美国航空公司 vs 谷歌利润率对比 |
| **案例拆解** | 公司/行业案例的成败分析 | 苹果、微软的垄断案例 |
| **原文精华** | 2-5 段原话，优先选含模型定义/数据结论/案例启示的段落 | "所有幸福的企业都是不同的……" |

#### Phase 4-C：精髓提炼

- **核心洞见**：一句金句
- **可操公式**：`{商业模型} = {变量A} × {变量B} / {变量C}` 或一个诊断矩阵

**示例（《从0到1》第三章）**：
- 核心洞见：**垄断企业才有真正的利润，竞争消灭利润**
- 可操公式：`企业价值 = 垄断程度 × 技术壁垒 × 网络效应 × 规模经济`

#### Phase 5-C：总结输出

1. 全书的核心商业模型汇编
2. 各模型的适用行业/阶段
3. 模型间的互补/矛盾关系
4. 批判性评价（是否过时/局限/文化差异）

---

### 分支 D：哲学/思想类

**典型书籍**：论语、存在与时间、沉思录

#### Phase 3-D：逐章拆解

| 要素 | 说明 | 示例（《论语·学而篇》） |
|:----:|:----:|:--------------------:|
| **核心命题** | 该章的核心主张/论断 | 学而时习之，不亦说乎 |
| **论证结构** | 如何论证（前提→推理→结论） | 学习→实践→愉悦感的递进关系 |
| **关键概念** | 独特术语及其含义 | "学""习""说"的独特定义 |
| **原文精华** | 2-5 段原话，优先选含核心命题/精妙论证的段落 | 原文段落 |

#### Phase 4-D：精髓提炼

- **核心洞见**：一句金句
- **可操公式**：`{命题} ↔ {前提1} + {前提2} ⇒ {结论}`（还原论证逻辑链条）

**示例（《论语·学而篇》）**：
- 核心洞见：**学习的真谛在于知行合一，在实践中获得愉悦**
- 可操公式：`学习的愉悦 = 接收新知 × 反复实践 ÷ 功利心态`

#### Phase 5-D：总结输出

1. 全书的问题树（作者要回答的核心问题 → 子问题）
2. 立场辨析（与同时代/不同时代思想家的对比）
3. 批判性审视（前提的合理性、论证有无漏洞）
4. 现代意义（该思想对今天的启示）

## 7. 关键约束

1. **原文引用必须忠实于原书**：所有引用的原话必须标注来源（章节/页数或来源平台），不得编造原文
2. **扉页必须保留**：报告必须包含书籍封面/扉页的核心一句话
3. **每章至少 2-5 段原文精华**：确保不是空洞的概括，而是有实质内容
4. **精髓提炼必须输出金句+公式**：不能只有金句没有公式，也不能只有公式没有金句
5. **类型兜底规则**：当无法判定书籍类型时，默认为实用/工具类
6. **信息标注**：在报告末尾注明信息来源（如"本文档基于豆瓣读书标注、公开书评摘录及原书公开信息整理"）
7. **冷门书籍说明**：当公开资料不足以完成完整拆解时，应在报告中说明局限，并给出用户自行补充阅读的建议
8. **去AI味处理**：除原书原文引用外，所有生成的内容（解析、概括、过渡、总结、口播稿等）必须按 [remove-ai-flavor-writing-skill](https://github.com/B1lli/remove-ai-flavor-writing-skill) 的规则去除AI写作痕迹，参考下方「去AI味速查表」

### 去AI味速查表

输出前逐项检查，确认不存在以下模式：

| 类别 | 高危险型 | 改写方向 |
|:----:|:---------|:---------|
| **二元对比壳** | `不是...而是` `并非...而是` `不在于...而在于` `不只是...更是` `与其说...不如说` | 如果 A 是铺垫，直接陈述 B；如果 A 和 B 都需要，写成具体关系而非说教式纠正 |
| **分步推进壳** | `先...再...` `第一步...第二步...` `从...到...` | 顺序改变结果才保留，否则直接写动作本身 |
| **本质断言** | `真正重要的是` `本质上` `核心在于` `底层逻辑` `关键就在于` | 直接说出主题本身，用事实/因果替代抽象强调 |
| **助手指路牌** | `下面我们来` `接下来` `我们可以看到` `总的来说` `值得注意的是` `说白了` `划重点` `拆一拆` | 直接进入内容，让内容自己推进 |
| **简化断言** | `很简单：` `答案很简单：` | 替换为具体条件、证据或决策过程 |
| **模版标点** | `概念：解释` `问题：答案` 等冒号模板句 | 只在自然需要的地方用冒号 |
| **过度平行** | 三段以上结构、长度、情绪一致的分句 | 砍掉一段，一段展开成例子，一段改为平实陈述 |
| **空结尾反问** | `你觉得呢？` `是不是很有启发？` `你有没有类似经历？` | 用事实、场景、后果收尾，或去掉 |

## 8. 输出格式 B — 口播稿

### 8.1 设计意图

在标准结构化报告（Section 5）之外，提供一种**可直接朗读的连续稿件格式**。适用于播客、音频节目、视频口播、读书直播等以听为主的场景。

### 8.2 转换规则

当用户要求"播报稿""朗读版""音频稿""连续输出"或明确指定使用本格式时，将原拆解内容按以下规则转换。**注意：转换完成后，必须执行去AI味检查（见 7. 去AI味速查表），确保口播稿中不存在AI痕迹。**

| 原结构化元素 | 转换方式 |
|:-----------:|---------|
| `#` / `##` / `###` 标题 | 转换为自然过渡句，用事实推进而非助手指路牌（不用"接下来我们来看看"这类） |
| `-` 列表 / 表格 | 转换为连续叙述，用"其中""包括""比如"引导 |
| `**加粗**` 强调 | 保留加粗，作为口语重音指示 |
| `>` 引用块 | 用"书里说""作者原话是"等自然引导 |
| 编号/序数词 | 用自然顺序表达，避免"第一、第二"的机械列举 |
| `{代码框}` 公式 | 用平实语言转述 |

### 8.3 核心写作原则

1. **无分段结构** — 不使用标题、列表、表格等结构化标记
2. **无旁白说明** — 不使用"（笑）""（停顿）""（主持人）"等舞台指示
3. **无显性编号** — 不使用"第一""第二""首先""其次""1."等机械分点
4. **自然过渡** — 段落之间用承上启下的语句连接，而非标题切换
5. **口语化** — 短句为主，使用"你"而非"读者"，避免长定语嵌套
6. **情绪节奏** — 开头有吸引力（设问/故事/反常识断言），中间有起伏，结尾有余韵
7. **开场有引、收尾有回** — 稿件最开头必须有问候+引出话题的**开场白**，最末尾必须有结束+回应的**结尾语**，两端形成完整的播报闭环
8. **去AI味** — 口播稿是人讲给人听的，不是AI写给AI读的。写完逐项检查去AI味速查表，清除二元对比壳、本质断言、助手指路牌、空结尾反问等AI常见模式

### 8.4 信息层次融合表

每一份连续内容稿件至少覆盖以下信息层次，但**全部以自然叙述方式呈现**：

| 信息层次 | 融合方式 |
|:--------:|---------|
| 个性开场 + 目标人群引导 | 第一段用"我是XX"建立身份+以"如果你是{目标人群}，{痛点}"的句式锁定听众 |
| 书籍元数据（书名/作者） | 紧接着自然带出 |
| 核心议题/全书主题 | 在前 1/3 处清楚点明 |
| 关键概念 | 在对应位置用通俗方式解释，避免生硬抛术语 |
| 实质案例/原文精华 | 用"书里提到""作者说"等自然引导 |
| 精髓提炼/核心洞见 | 在稿件高潮处着重呈现，适当放慢节奏 |
| 批判性审视 | 收尾时自然融入，不单独成段 |

### 8.5 写作模板

```
━━━ 开场白 ━━━
{个性化问候：如"各位朋友大家好，我是你们的老朋友XX"}
{目标人群 + 痛点铺开：一句话勾勒目标人群，快速带出书中能解决的问题}
{引出书名+作者}
{开篇钩子：一个反常识断言 / 引人深思的问题 / 生动的故事场景}

自然地引出全书核心议题——这本书在探讨什么，为什么值得关注

逐章或按话题群组介绍关键内容：
  - 每个概念用日常语言解释
  - 引用案例或实验时用"书中的XX故事说""作者指出"
  - 关键结论处适当放慢节奏

全书的精髓提炼——最值得记住的观点 / 公式 / 模型

批判性审视——哪些观点有局限、哪些需要结合个人情况判断

自然过渡到结语

━━━ 结尾语 ━━━
{对整期内容的简短回顾}
{开放性余韵 / 引人深思的问题 / 鼓励行动的一句话}
{结束语：如"感谢你的收听，我们下期再见"}

━━━ 去AI味检查 ━━━
▢ 无二元对比壳（不是...而是/并不在于...而在于）
▢ 无本质断言（真正重要的是/本质上/核心在于）
▢ 无分步推进壳（先...再...）
▢ 无助手指路牌（总的来说/值得注意的是/划重点）
▢ 无简化断言（很简单：/答案很简单：）
▢ 无模板标点（概念：解释）
▢ 无空结尾反问
▢ 段落长短不一，节奏有起伏
```

### 8.6 写作示例（基于《精进》第一章，含开场白和结尾语，已去AI味）

各位朋友大家好，我是你们的老朋友XX。如果你是创业者、管理者，或者一个正在快速变化里追赶的人——今天这本书能帮你找到那个卡住你的点。它叫《精进》，作者采铜，浙江大学心理学博士。他在扉页上写了一句很扎心的话——盲目的努力，只是一种缓慢的叠加。采铜用一种很特别的方式开了头，他讲了老家的一个挂钟。每次回家看到那钟，他就觉得时间走得很慢，很放松，稿子没写就没写吧，慢慢来。但用闹钟看时间的时候，就觉得时间走得好急。他把这个叫"母亲的挂钟效应"——你对时间的感觉不是客观的，心理状态和环境会改变它。焦虑的时候时间紧迫，放松的时候时间变慢。这引出了第一章的核心问题：你跟时间的关系，决定了你怎么度过这一生。

采铜提了一个叫"郑重"的概念，说郑重不是板着脸做计划，而是一种对当下事情全身心投入的状态，不敷衍、不迟疑、不摇摆。然后他给了一个判断框架，叫"采铜法则"，看两样：一件事能带来多大收益，收益能持续多久。他建议多做事后效果还能持续很久的事，哪怕单次收益不高。每天背十首宋词，单次收益不大，一年下来底蕴就不一样了。反过来，那些做的时候很快乐、关上屏幕什么都不剩的事——比如刷短视频——能少就少。下次纠结做什么的时候，问问自己：做完之后，效果明天还在吗？以上就是今天关于时间管理的内容，如果对你有用，就从今晚开始做一件长半衰期的小事吧。感谢收听，下期再见。
